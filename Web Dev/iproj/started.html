Sketch Layout

Programs for Arduino are called sketches. Sketches are broken down into three parts: Variable Initialization, setup, and loop. The first thing that needs to be done is variable initialization. In this section of the sketch variables are declared, and they are initialized to their beginning values. In the second phase, setup, the functionality of the pins is cleared. All the pins that will used will be declared as input or output. Finally, in the loop phase, the bulk of the program is ran. As the name suggests, loops continually runs until the arduino is shut off. After the loop method, helper methods can be written to improve the readability of the loop method.

First Sketch - Blink

The Arduino IDE comes with some sketches already preprogramed. They can be found under file then examples. The easiest program to demonstrate the functionality of Arduino is the Blink program. It can be found in the examples under 01. Basics. Here you can see that there are no variables declared. However; pin 13 could have been declared in a variable such as int ledPin = 13; Then where ever 13 is in Blink, it could be replaced by the variable name for readability. Once the program is uploaded below you should see the small yellow led next to pin 13 blink. An extra led has been added in the video below for added effect.
